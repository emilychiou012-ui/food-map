# PRD：美食訂位小幫手
**Product Requirements Document**
版本：v1.0 ｜ 日期：2026-06-09 ｜ 作者：Emily Chiou

---

## 1. 產品概覽

### 1.1 產品名稱
美食訂位小幫手（Food Booking Assistant）

### 1.2 產品簡介
一款專為職場團隊設計的靜態網頁應用，用於管理周邊餐廳資訊、記錄訂位歷程，並支援多人場合的餐廳挑選與篩選。無需後端或登入機制，所有資料儲存於瀏覽器的 localStorage，可直接部署於 GitHub Pages 使用。

### 1.3 核心價值主張
> 「午餐・下午茶・聚餐・一鍵搞定」

---

## 2. 問題背景

### 2.1 痛點
- 辦公室同仁需要安排部門聚餐、商務宴客、平日午餐，每次都需要重新查找餐廳資訊
- 餐廳電話、訂位網址、營業時間散落在各種 LINE 訊息、截圖與 Google 搜尋中
- 不同場合（素食友善、商務聚餐、同事生日）需要不同類型的餐廳，難以快速篩選
- 沒有一個集中的地方記錄「這間不錯值得回購」或「這間踩雷」的心得

### 2.2 解決方案
建立一個個人化的餐廳資料庫，支援新增、編輯、篩選、訂位記錄與心得管理，讓每次找餐廳都能在 30 秒內完成決策。

| 角色 | 描述 |
|------|------|

## 4. 功能需求

### 4.1 前導頁面（Landing Page）

**目的：** 讓使用者在進入主畫面前，先依據地區縮小範圍。

| 功能 | 說明 |
|------|------|
| 區域選擇 | 南港區、內湖區、信義區、中山區，或全部瀏覽 |
| 即時計數 | 每個區域顯示已收錄的餐廳數量 |
| 進場動畫 | 使用 GSAP 製作 Logo → 標題 → 副標 → 分隔線 → 提示 → 區域卡片的逐一淡入效果 |
| 退場動畫 | 選擇區域後，頁面向上淡出（GSAP fade + y offset） |
| 背景圖 | 使用者自行提供的巴黎咖啡廳風格照片（`bg.jpg`），疊加半透明遮罩 |
| 切換區域 | 主畫面右上角保留「切換區域」按鈕，可隨時返回前導頁 |

**區域卡片設計：**

| 區域 | 食物 Emoji 主題 | 標語 |

| 南港區 | 🥐🍳☕🫐 | 展覽商圈・南港車站 |
| 內湖區 | 🦆🍜🥢🍱 | 內科商圈・碧湖公園 |

### 4.2 餐廳管理

#### 4.2.1 新增餐廳表單欄位

表單欄位依照詳情面板顯示順序排列：

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 店名 | 文字 | ✅ | 餐廳名稱 |
| 用餐時段 | 下拉選單 | ✅ | 早餐 / 午餐 / 下午茶 / 晚餐 / 早午餐 / 聚餐 |
| 適合場合 | 下拉選單 | — | 部門聚餐 / 商務聚餐 / 同事生日 / 平日午餐 / 不限 |
| 飲食需求 | 下拉選單 | — | 葷食為主 / 素食友善 / 全素・純素 |
| 料理類型 | 下拉選單 | — | 台灣 / 日式 / 韓式 / 中式 / 西式 / 義式 / 泰式 / 咖啡甜點 / 火鍋燒烤 / 便當自助 / 其他 |
| 電話 | 電話輸入 | — | 支援 `tel:` 連結一鍵撥號 |
| 人均消費（元） | 數字區間 | — | 最低 ～ 最高，顯示為 `$350 - $600` |
| 營業時間 | 文字 | — | 自由格式，例：11:00-14:30，週一公休 |
| 付款方式 | 多選核取 | — | 現金 / Visa / Mastercard / JCB / AE Card |
| 地址 | 文字 | — | 顯示 Google Maps 嵌入地圖 |
| 評分 | 星星點選 | — | 1–5 顆星互動式評分 |
| 需要訂位 | 單選 | — | 不需要 / 建議訂位 / 必須訂位 |
| 外送方式 | 單選 | — | 僅限自取 / 可外送 / Uber Eats / Foodpanda |
| 餐廳網頁 | URL | — | 官網 / 食評 / Google Map，顯示為「餐廳介紹」按鈕 |
| 訂位網址 | URL | — | 顯示為「立即訂位」按鈕 |
| 餐廳特色備註 | 多行文字 | — | 必點菜色、停車、氛圍等，以橘色區塊顯示 |

#### 4.2.2 編輯與刪除
- 點選左側餐廳卡片可在右側詳情面板查看完整資訊
- 詳情面板右上角提供「✏️ 編輯」與「🗑️ 刪除」按鈕
- 刪除前跳出確認對話框防止誤刪

### 4.3 篩選功能

篩選列提供以下維度，可同時複合篩選：

| 篩選項目 | 選項 |
|----------|------|
| 搜尋店名 | 即時關鍵字搜尋 |
| 餐點種類 | 早餐 / 午餐 / 下午茶 / 晚餐 / 早午餐 / 聚餐 |
| 適合場合 | 部門聚餐 / 商務聚餐 / 同事生日 / 平日午餐 |
| 外送方式 | 僅限自取 / 可外送 / Uber Eats / Foodpanda |
| 飲食需求 | 葷食為主 / 素食友善 / 全素・純素 |

篩選結果即時顯示「共 N 筆」計數。

### 4.4 詳情面板

詳情面板資訊顯示順序與表單欄位完全對應：

```
[標籤列] 用餐時段 | 場合 | 飲食需求 | 料理 | 外送 | 訂位
[星星評分]
電話         → 支援一鍵撥號 + 複製按鈕
營業時間     → 純文字
人均消費     → $min - $max 格式
付款方式     → 標籤呈現，現金 / 刷卡分色
地址         → 文字 + 複製 + Google Maps 嵌入
[餐廳介紹 按鈕]  [立即訂位 按鈕]
✨ 餐廳特色備註（橘色區塊）
💬 個人心得（回購理由 / 黑名單）← 詳情面板專屬欄位
```

**兩種備註欄位的區別：**

| 欄位 | 來源 | 用途 |
|------|------|------|
| 餐廳特色備註 | 新增/編輯表單填入 | 餐廳客觀資訊：必點、停車、氛圍 |
| 個人心得 | 詳情面板直接儲存 | 主觀評價：值得回購 / 黑名單理由 |

## 5. 技術規格

### 5.1 技術架構

| 項目 | 選用技術 |
|------|----------|
| 框架 | 純 HTML / CSS / JavaScript（無框架） |
| 資料儲存 | 瀏覽器 `localStorage` |
| 字型 | Google Fonts：Noto Sans TC + Noto Serif TC |
| 動畫 | GSAP 3.12.5（CDN 引入） |
| 地圖 | Google Maps Embed API（iframe） |
| 部署 | GitHub Pages（靜態托管） |

### 5.2 檔案結構

```
food-map/
├── index.html           # 主頁面結構
├── style.css            # 完整樣式設計系統
├── app.js               # 主要業務邏輯
├── import-restaurants.js # 預設餐廳資料 + 匯入函式
├── bg.jpg               # 背景照片（使用者自行提供）
└── PRD.md               # 本文件
```

### 5.3 資料結構（單筆餐廳物件）

```javascript
{
  id: Number,            // Date.now() 生成
  name: String,          // 店名
  mealType: String,      // 用餐時段
  occasion: String,      // 適合場合
  dietary: String,       // 飲食需求
  category: String,      // 料理類型
  phone: String,
  priceMin: String,      // 人均消費下限
  priceMax: String,      // 人均消費上限
  address: String,
  hours: String,         // 營業時間
  rating: Number,        // 0–5
  reservation: String,   // 需要訂位
  visited: String,       // 外送方式
  payment: Array,        // 付款方式（多選）
  restaurantUrl: String,
  bookingUrl: String,
  notes: String,         // 餐廳特色備註
  remark: String,        // 個人心得（詳情面板填寫）
  nextBooking: Object,   // 下次訂位資訊（預留）
  bookingHistory: Array, // 訂位歷史紀錄
  createdAt: String
}
```

## 7. 資料維護機制

### 7.1 自動資料補齊（patchRestaurantData）

初始化時執行一次性 patch，針對已存入 localStorage 的餐廳補齊以下欄位（僅在原欄位空白時更新）：
- 營業時間（`hours`）
- 電話（`phone`）
- 地址（`address`，長度 < 10 視為無效）

### 7.2 舊資料相容性

若 localStorage 中存有舊版資料（如 `visited: '想去'`），透過 console 遷移腳本統一改為 `'僅限自取'`。

## 10. 限制與已知問題

| 項目 | 說明 |
|------|------|
| 資料儲存 | 使用 localStorage，清除瀏覽器資料後遺失 |
| 多裝置同步 | 目前不支援，資料僅存於單一瀏覽器 |
| 背景圖片 | 需與 HTML 檔案放在同一目錄（`bg.jpg`） |
| Google Maps | 嵌入地圖需要網路連線才能顯示 |
| 地址精準度 | 部分餐廳地址為近似位置，可能影響地圖定位 |
